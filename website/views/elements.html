    • <assign>

    The assign element assigns a value to a variable.

    Examples

    <!-- Assign Strings -->
<assign id="lead" value="Luke"/>
<assign id="princess" value="Leia"/>

<!-- Set "heroes" to contain "Luke, Leia, and Han" via substitution -->
<assign id="heroes" value="${hero}, ${princess}, and Han"/>

<!-- Perform math via JEXL and variable substitution -->
<assign id="domesticBoxOffice" value="460998007"/>
<assign id="worldBoxOffice" value="797900000"/>
<assign id="boxOfficeTotal" value="${domesticBoxOffice} + ${worldBoxOffice}"/>

<!-- Instantiate instance of the 'ex.Person' class, with JEXL's 'new' command to invoke a two-argument -->
<!-- constructor. In Java, this would translate to "person = new ex.Person( 'Darth', 'Vader' );" -->
<assign id="person" value="new( ex.Person, 'Darth', 'Vader' )"/>

    <assign> Attribute Reference

    Name Presence Description
    id Required ID (aka name) of the variable to assign.
    value Required Value to assign to the variable.
    if Optional JEXL expression that prevents assignment if it evaluates to false. Otherwise, the assignment proceeds as normal.

    <commit>

    The commit element commits the current transaction on the specified connection object. An ExecutionException is thrown if the connection cannot be found under the given ID.

    Examples

    <!-- Open a database connection from the DataSource -->
<connection id="myConn" datasource-id="MyDatasource" autocommit="false"/>

... SQL actions not shown ...

<!-- Commit the transaction -->
<commit id="myConn"/>

    <commit> Attribute Reference

    Name Presence Description
    id Required ID of the database connection on which to commit the current transaction.

    <connection>

    Open a database connection from the DataSource based on a standard Java DataSource.

    Examples

    <!-- Open a database connection from the DataSource -->
<connection id="myConn" datasource-id="MyDatasource" autocommit="false"/>

<!-- Create another -->
<connection id="connection2" datasource-id="OtherDatasource" autocommit="true"/>

    <connection> Attribute Reference

    Name Presence Description
    id Required ID of the database connection to create.
    datasource-id Required ID of the DataSource from which to retrieve the connection.
    autocommit Optional Boolean indicating if the connection should be set to autocommit. Valid values are "true" and "false". Default value is defined by the DataSource.

    <if>

    Execute the enclosed block of code if the given condition is true.

    The <if> action can contain any of the following elements:

    • assign
    • close
    • commit
    • connection
    • exit
    • if
    • include
    • objectFactory
    • properties
    • query
    • rollback
    • sql
    • script

    Examples

    <!-- Default commission rate to 10% -->
<assign id="commissionRate" value="0.10"/>

<!-- Sales over 2000 doubles the salesperson's commission rate -->
<if condition="salesTotal > 2000">
    <assign id="commissionRate" value="${commissionRate} * 2.0"/>

    <!-- 'if' actions can be nested. Let's use it to double the commission rate again. --!>
    <if condition="salesTotal > 3000 and surveysReturned > 2">
        <assign id="commissionRate" value="${commissionRate} * 2.0"/>
    </if >
</if >

<!-- Calculate commission -->
<assign id="commission" value="${salesTotal} & ${commissionRate}"/>

    <if> Attribute Reference

    Name Presence Description
    condition Required Condition to evaluate. There is no need for the ${..} wrappers around variables, since the entire condition is passed to JEXL for evaluation.
